Algoritmos e Lógica

InicianteImplementa lógica simples, sem se preocupar com eficiência▼
exemplo de código
C#
// Verificar se número é primo — abordagem ingênua
public bool IsPrimo(int n)
{
    for (int i = 2; i < n; i++)
        if (n % i == 0) return false;
    return true;
}
// O(n) — testa todos os divisores até n
Sinal: Não conhece que basta testar até √n. Loop desnecessário percorre toda a base.
BásicoResolve problemas conhecidos, percebe casos extremos▼
exemplo de código
C#
public bool IsPrimo(int n)
{
    if (n < 2) return false;
    if (n == 2) return true;
    if (n % 2 == 0) return false;
    for (int i = 3; i < n; i += 2)
        if (n % i == 0) return false;
    return true;
}
// Melhora: pula pares. Mas ainda não chegou em √n
Positivo: Percebe casos extremos (n < 2, par). Ainda falta conhecimento de complexidade formal.
IntermediárioConhece complexidade e aplica soluções eficientes▼
exemplo de código
C#
// O(√n) — testando até a raiz quadrada
public bool IsPrimo(int n)
{
    if (n < 2) return false;
    if (n == 2) return true;
    if (n % 2 == 0) return false;
    int limite = (int)Math.Sqrt(n);
    for (int i = 3; i <= limite; i += 2)
        if (n % i == 0) return false;
    return true;
}
// Consegue explicar POR QUÊ isso funciona
Marco: Explica a prova: se n tem divisor maior que √n, o complementar já seria menor e teria sido encontrado antes.
AvançadoEscolhe o algoritmo certo conforme o contexto▼

// Crivo de Eratóstenes — O(n log log n) para múltiplos números
public bool[] GerarPrimos(int limite)
{
    var ehPrimo = new bool[limite + 1];
    Array.Fill(ehPrimo, true);
    ehPrimo[0] = ehPrimo[1] = false;
    for (int i = 2; i * i <= limite; i++)
        if (ehPrimo[i])
            for (int j = i * i; j <= limite; j += i)
                ehPrimo[j] = false;
    return ehPrimo;
}
// Escolhe Crivo quando precisa de MÚLTIPLAS consultas
Diferencial: "Preciso checar uma vez ou mil vezes?" muda o algoritmo ideal.

// Miller-Rabin — teste probabilístico para números enormes (criptografia)
public static bool IsProbablyPrime(long n, int rounds = 10)
{
    if (n < 2) return false;
    if (n == 2 || n == 3) return true;
    if (n % 2 == 0) return false;
    long d = n - 1; int r = 0;
    while (d % 2 == 0) { d /= 2; r++; }
    var rng = new Random();
    for (int i = 0; i < rounds; i++)
    {
        long a = 2 + (long)(rng.NextDouble() * (n - 4));
        long x = ModPow(a, d, n);
        if (x == 1 || x == n - 1) continue;
        bool composto = true;
        for (int j = 0; j < r - 1; j++) { x = ModMul(x, x, n); if (x == n - 1) { composto = false; break; } }
        if (composto) return false;
    }
    return true;
}
// Discute: quando determinístico vs probabilístico, relação com RSA
Especialista: Conhece as fronteiras do problema. Sabe quando usar determinístico (bases fixas para n < 3.2×10¹⁸) e explica o trade-off com RSA.

// LRU Cache — O(1) para get e put (Dictionary + LinkedList)
public class LruCache<TKey, TValue>
{
    private readonly int _capacity;
    private readonly Dictionary<TKey, LinkedListNode<(TKey, TValue)>> _map;
    private readonly LinkedList<(TKey Key, TValue Value)> _list;
    public LruCache(int capacity) { _capacity = capacity; _map = new(capacity); _list = new(); }
    public TValue Get(TKey key)
    {
        if (!_map.TryGetValue(key, out var node)) return default;
        _list.Remove(node); _list.AddFirst(node);
        return node.Value.Value;
    }
    public void Put(TKey key, TValue value)
    {
        if (_map.TryGetValue(key, out var node)) { _list.Remove(node); _map.Remove(key); }
        else if (_map.Count >= _capacity) { _map.Remove(_list.Last.Value.Key); _list.RemoveLast(); }
        _map[key] = _list.AddFirst((key, value));
    }
}
// Especialista combina estruturas para atingir complexidade ótima
Especialista: Combina estruturas para atingir complexidade ótima. Entende impacto no GC e discute alternativas como ArrayPool para hot paths.

AvançadoModelagem rica — invariantes protegidos pela própria entidade▼
exemplo de código

// Modelo rico com invariantes protegidos
public class Pedido
{
    private readonly List<ItemPedido> _itens = new();
    public IReadOnlyList<ItemPedido> Itens => _itens.AsReadOnly();
    public StatusPedido Status { get; private set; } = StatusPedido.Aberto;
    public decimal Total => _itens.Sum(i => i.Subtotal);

    public void AdicionarItem(Produto produto, int quantidade)
    {
        if (Status != StatusPedido.Aberto)
            throw new InvalidOperationException("Pedido não está aberto.");
        if (quantidade <= 0)
            throw new ArgumentException("Quantidade deve ser positiva.");
        var existente = _itens.FirstOrDefault(i => i.ProdutoId == produto.Id);
        if (existente != null) existente.IncrementarQuantidade(quantidade);
        else _itens.Add(new ItemPedido(produto, quantidade));
    }

    public void Fechar()
    {
        if (!_itens.Any()) throw new InvalidOperationException("Pedido sem itens.");
        Status = StatusPedido.Fechado; // só muda por aqui
    }
}
Marco: A entidade é responsável pelos próprios invariantes. Impossível colocar em estado inválido de fora.

InicianteSELECT * e filtra na aplicação — full scan em produção▼

IntermediárioModelagem eficiente, índices, paginação server-side▼
exemplo de código
SQL
SELECT p.Id, p.Total, p.CriadoEm, c.Nome, COUNT(i.Id) AS TotalItens
FROM Pedidos p
INNER JOIN Clientes c ON c.Id = p.ClienteId
INNER JOIN ItensPedido i ON i.PedidoId = p.Id
WHERE p.Status = 'Aberto'
  AND p.CriadoEm >= DATEADD(DAY, -30, GETDATE())
GROUP BY p.Id, p.Total, p.CriadoEm, c.Nome
ORDER BY p.CriadoEm DESC
OFFSET 0 ROWS FETCH NEXT 20 ROWS ONLY;

AvançadoPlano de execução, Window Functions, diagnóstico de queries lentas▼
exemplo de código

-- Window Function — evita subquery correlacionada
SELECT
    p.Id, p.Total,
    SUM(p.Total) OVER (PARTITION BY p.ClienteId
                       ORDER BY p.CriadoEm
                       ROWS UNBOUNDED PRECEDING) AS TotalAcumulado,
    ROW_NUMBER() OVER (PARTITION BY p.ClienteId
                       ORDER BY p.Total DESC) AS RankPorCliente
FROM Pedidos p;

-- Diagnóstico de queries lentas
SELECT TOP 10
    qs.total_logical_reads / qs.execution_count AS avg_reads,
    qs.total_elapsed_time / qs.execution_count / 1000 AS avg_ms,
    SUBSTRING(qt.text, 1, 200) AS query_text
FROM sys.dm_exec_query_stats qs
CROSS APPLY sys.dm_exec_sql_text(qs.sql_handle) qt
ORDER BY avg_ms DESC;

public class PedidoAggregate
{
    private readonly List<IDomainEvent> _uncommitted = new();
    public StatusPedido Status { get; private set; }
    public decimal Total { get; private set; }

    public static PedidoAggregate From(IEnumerable<IDomainEvent> eventos)
    {
        var agg = new PedidoAggregate();
        foreach (var e in eventos) agg.Apply(e); // replay
        return agg;
    }
    public void Fechar()
    {
        if (Status != StatusPedido.Aberto) throw new DomainException("Não está aberto.");
        var e = new PedidoFechadoEvent(Id, DateTime.UtcNow);
        Apply(e);
        _uncommitted.Add(e); // persiste o EVENTO, não o estado
    }
    private void Apply(IDomainEvent e)
    {
        if (e is PedidoFechadoEvent) Status = StatusPedido.Fechado;
        if (e is ItemAdicionadoEvent item) Total += item.Subtotal;
    }
}
