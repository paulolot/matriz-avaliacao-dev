Algoritmos e Lógica
loops recursão complexidade otimização
Iniciante Implementa lógica simples, sem se preocupar com eficiência ▼
exemplo de código
C#
// Verificar se número é primo — abordagem ingênua
public bool IsPrimo(int n)
{
    for (int i = 2; i < n; i++)
    {
        if (n % i == 0) return false;
    }
    return true;
}
// Problema: O(n) — testa todos os divisores até n
Sinal de alerta: Não conhece que basta testar até √n. Loops desnecessários em toda a base.
Básico Resolve problemas conhecidos, começa a pensar em casos extremos ▼
exemplo de código
C#
// Mesmo problema, mais consciente dos casos base
public bool IsPrimo(int n)
{
    if (n < 2) return false;
    if (n == 2) return true;
    if (n % 2 == 0) return false;

    for (int i = 3; i < n; i += 2)
    {
        if (n % i == 0) return false;
    }
    return true;
}
// Melhora: pula pares. Mas ainda não chegou em √n
Positivo: Percebe casos extremos (n < 2, par). Ainda falta conhecimento de complexidade formal.
Intermediário Conhece complexidade e aplica soluções eficientes ▼
exemplo de código
C#
// O(√n) — testando apenas até a raiz quadrada
public bool IsPrimo(int n)
{
    if (n < 2) return false;
    if (n == 2) return true;
    if (n % 2 == 0) return false;

    int limite = (int)Math.Sqrt(n);
    for (int i = 3; i <= limite; i += 2)
    {
        if (n % i == 0) return false;
    }
    return true;
}
// Consegue explicar POR QUÊ isso funciona
Marco: Consegue explicar a prova matemática — se n tem um divisor maior que √n, o outro é menor, e já teria sido encontrado.
Avançado Analisa trade-offs e escolhe algoritmo certo para o contexto ▼
exemplo de código
C#
// Crivo de Eratóstenes — O(n log log n) para múltiplos números
public bool[] GerarPrimos(int limite)
{
    var ehPrimo = new bool[limite + 1];
    Array.Fill(ehPrimo, true);
    ehPrimo[0] = ehPrimo[1] = false;

    for (int i = 2; i * i <= limite; i++)
    {
        if (ehPrimo[i])
        {
            for (int j = i * i; j <= limite; j += i)
                ehPrimo[j] = false;
        }
    }
    return ehPrimo;
}
// Escolhe Crivo quando precisa de MÚLTIPLAS consultas
Diferencial: Sabe que a resposta muda conforme o problema. "Preciso checar uma vez ou mil vezes?" muda o algoritmo ideal.
Especialista Resolve problemas com restrições severas, define abordagem para o time ▼
exemplo de código
C#
// Miller-Rabin — teste probabilístico, para números enormes
public static bool IsProbablyPrime(long n, int rounds = 10)
{
    if (n < 2) return false;
    if (n == 2 || n == 3) return true;
    if (n % 2 == 0) return false;

    long d = n - 1;
    int r = 0;
    while (d % 2 == 0) { d /= 2; r++; }

    var rng = new Random();
    for (int i = 0; i < rounds; i++)
    {
        long a = 2 + (long)(rng.NextDouble() * (n - 4));
        long x = ModPow(a, d, n);
        if (x == 1 || x == n - 1) continue;

        bool composto = true;
        for (int j = 0; j < r - 1; j++)
        {
            x = ModMul(x, x, n);
            if (x == n - 1) { composto = false; break; }
        }
        if (composto) return false;
    }
    return true; // provavelmente primo
}
// Discute probabilidade de erro, quando usar determinístico vs probabilístico
Especialista: Conhece as fronteiras do problema. Sabe quando usar determinístico (Miller-Rabin com bases fixas para n < 3.2×10¹⁸) e explica o trade-off com RSA.

Estrutura de Dados
ListDictionary QueueHashSetmemória
Iniciante Usa List para tudo, sem conhecer outras estruturas ▼
exemplo de código
C#
// Verificar duplicatas com List — O(n²)
var usuarios = new List<string> { "ana", "bob", "ana", "carlos" };
var duplicatas = new List<string>();

for (int i = 0; i < usuarios.Count; i++)
{
    for (int j = i + 1; j < usuarios.Count; j++)
    {
        if (usuarios[i] == usuarios[j] && !duplicatas.Contains(usuarios[i]))
            duplicatas.Add(usuarios[i]);
    }
}
// Usa lista quando deveria usar Set
Problema: O(n²) desnecessário. Não conhece HashSet, que resolveria em O(n).
Intermediário Escolhe a estrutura certa para o problema ▼
exemplo de código
C#
// HashSet para unicidade — O(n)
var vistos = new HashSet<string>();
var duplicatas = new HashSet<string>();

foreach (var usuario in usuarios)
{
    if (!vistos.Add(usuario))
        duplicatas.Add(usuario);
}

// Dictionary para contar frequência — O(n)
var frequencia = new Dictionary<string, int>();
foreach (var u in usuarios)
{
    frequencia.TryGetValue(u, out int count);
    frequencia[u] = count + 1;
    // ou: frequencia[u] = frequencia.GetValueOrDefault(u) + 1;
}
// Conhece quando usar cada estrutura
Marco: Raciocina sobre complexidade antes de escolher. HashSet para presença, Dictionary para frequência, Queue para fila, Stack para pilha.
Especialista Implementa estruturas customizadas e analisa memória ▼
exemplo de código
C#
// LRU Cache — estrutura composta (Dictionary + LinkedList)
public class LruCache<TKey, TValue>
{
    private readonly int _capacity;
    private readonly Dictionary<TKey, LinkedListNode<(TKey, TValue)>> _map;
    private readonly LinkedList<(TKey Key, TValue Value)> _list;

    public LruCache(int capacity)
    {
        _capacity = capacity;
        _map = new Dictionary<TKey, LinkedListNode<(TKey, TValue)>>(capacity);
        _list = new LinkedList<(TKey, TValue)>();
    }

    public TValue Get(TKey key)
    {
        if (!_map.TryGetValue(key, out var node)) return default;
        _list.Remove(node);
        _list.AddFirst(node);
        return node.Value.Value;
    }

    public void Put(TKey key, TValue value)
    {
        if (_map.TryGetValue(key, out var node))
        {
            _list.Remove(node);
            _map.Remove(key);
        }
        else if (_map.Count >= _capacity)
        {
            var last = _list.Last;
            _map.Remove(last.Value.Key);
            _list.RemoveLast();
        }
        var newNode = _list.AddFirst((key, value));
        _map[key] = newNode;
    }
}
// O(1) para get e put — estrutura combinada para resolver o problema
Especialista: Combina estruturas para atingir complexidade ótima. Entende impacto no GC (.NET) e discute alternativas como ArrayPool<T> para hot paths.

// Modelo anêmico — anti-pattern clássico
public class Pedido
{
    public int Id { get; set; }
    public decimal Total { get; set; }
    public string Status { get; set; }
    public List<Item> Itens { get; set; }
}

// Toda a lógica fica espalhada em Services externos
public class PedidoService
{
    public void Fechar(Pedido pedido)
    {
        if (pedido.Total > 0 && pedido.Itens.Count > 0)
            pedido.Status = "Fechado";
    }
}
// Pedido não protege seus próprios invariantes
Problema: Qualquer parte do sistema pode setar pedido.Status = "Cancelado" diretamente, sem validação. Invariantes não são protegidos.
Avançado Modelagem rica — comportamento e proteção de invariantes ▼
exemplo de código
C#
// Modelo rico com invariantes protegidos
public class Pedido
{
    private readonly List<ItemPedido> _itens = new();
    public IReadOnlyList<ItemPedido> Itens => _itens.AsReadOnly();
    public StatusPedido Status { get; private set; } = StatusPedido.Aberto;
    public decimal Total => _itens.Sum(i => i.Subtotal);

    public void AdicionarItem(Produto produto, int quantidade)
    {
        if (Status != StatusPedido.Aberto)
            throw new InvalidOperationException("Pedido não está aberto.");
        if (quantidade <= 0)
            throw new ArgumentException("Quantidade deve ser positiva.");

        var itemExistente = _itens.FirstOrDefault(i => i.ProdutoId == produto.Id);
        if (itemExistente != null)
            itemExistente.IncrementarQuantidade(quantidade);
        else
            _itens.Add(new ItemPedido(produto, quantidade));
    }

    public void Fechar()
    {
        if (!_itens.Any())
            throw new InvalidOperationException("Pedido sem itens.");
        Status = StatusPedido.Fechado;
    }
}
// Status só muda por métodos — invariantes garantidos
Marco: A entidade é responsável por seus próprios invariantes. Ninguém de fora pode colocar o pedido em um estado inválido.
Especialista Value Objects, Aggregates, Domain Events ▼
exemplo de código
C#
// Value Object — imutável, igualdade por valor
public record Dinheiro(decimal Valor, string Moeda)
{
    public static Dinheiro operator +(Dinheiro a, Dinheiro b)
    {
        if (a.Moeda != b.Moeda)
            throw new InvalidOperationException("Moedas diferentes.");
        return new Dinheiro(a.Valor + b.Valor, a.Moeda);
    }
    public static Dinheiro Zero(string moeda) => new(0, moeda);
}

// Aggregate Root com Domain Event
public class Pedido : AggregateRoot
{
    private readonly List<ItemPedido> _itens = new();
    public Dinheiro Total => _itens.Aggregate(
        Dinheiro.Zero("BRL"),
        (acc, item) => acc + item.Subtotal);

    public void Fechar()
    {
        if (!_itens.Any())
            throw new DomainException("Pedido sem itens não pode ser fechado.");

        Status = StatusPedido.Fechado;
        AddDomainEvent(new PedidoFechadoEvent(Id, Total));
    }
}
// Domain Event notifica outros bounded contexts de forma desacoplada
Especialista: Domina DDD tático. Discute quando DDD é overkill (CRUD simples) e quando é essencial (domínio complexo com regras de negócio ricas).

public async Task<IEnumerable<PedidoDto>> BuscarAbertosAsync(

Problema: SELECT * transfere colunas desnecessárias. Filtro na aplicação em vez do banco. Em 1M de registros, isso derruba o sistema.
Intermediário Joins, índices, paginação, consultas projetadas ▼
exemplo de código
SQL
-- Consulta eficiente com JOIN, paginação e projeção
SELECT
    p.Id,
    p.Total,
    p.CriadoEm,
    c.Nome AS Cliente,
    COUNT(i.Id) AS TotalItens
FROM Pedidos p
INNER JOIN Clientes c ON c.Id = p.ClienteId
INNER JOIN ItensPedido i ON i.PedidoId = p.Id
WHERE p.Status = 'Aberto'
  AND p.CriadoEm >= DATEADD(DAY, -30, GETDATE())
GROUP BY p.Id, p.Total, p.CriadoEm, c.Nome
ORDER BY p.CriadoEm DESC
OFFSET 0 ROWS FETCH NEXT 20 ROWS ONLY;

-- Índice composto para suportar esse filtro
CREATE INDEX IX_Pedidos_Status_CriadoEm
ON Pedidos (Status, CriadoEm DESC)
INCLUDE (Total, ClienteId);
Marco: Sabe criar índices alinhados com a query. Entende INCLUDE para evitar key lookup. Paginação server-side.
Especialista Plano de execução, particionamento, estratégia de dados ▼
exemplo de código
SQL
-- Diagnóstico de query lenta com plano de execução
SET STATISTICS IO, TIME ON;

SELECT * FROM sys.dm_exec_query_stats
CROSS APPLY sys.dm_exec_sql_text(sql_handle)
WHERE total_elapsed_time / execution_count > 1000000  -- avg > 1s
ORDER BY total_logical_reads / execution_count DESC;

-- Window Function para evitar subquery correlacionada
SELECT
    p.Id,
    p.Total,
    SUM(p.Total) OVER (PARTITION BY p.ClienteId
                       ORDER BY p.CriadoEm
                       ROWS UNBOUNDED PRECEDING) AS TotalAcumulado,
    ROW_NUMBER() OVER (PARTITION BY p.ClienteId
                       ORDER BY p.Total DESC) AS RankPorCliente
FROM Pedidos p;

-- Columnstore Index para analytics (leitura de milhões de linhas)
CREATE COLUMNSTORE INDEX CIX_Pedidos_Analytics
ON Pedidos (CriadoEm, Total, Status, ClienteId);
Especialista: Lê plano de execução como código. Discute OLTP vs OLAP, quando usar columnstore, quando partir a tabela, e como medir impacto real.

// God method — viola SRP, Open/Closed e DRY
public async Task ProcessarPedido(int pedidoId, string tipoDesconto)
{
    var conn = new SqlConnection("Server=prod;Database=db;...");
    conn.Open();
    var pedido = conn.QuerySingle<Pedido>(
        "SELECT * FROM Pedidos WHERE Id = @id", new { id = pedidoId });

    decimal desconto = 0;
    if (tipoDesconto == "VIP") desconto = 0.20m;
    else if (tipoDesconto == "PROMO") desconto = 0.10m;
    else if (tipoDesconto == "FUNCIONARIO") desconto = 0.30m;

    pedido.Total -= pedido.Total * desconto;
    pedido.Status = "Processado";

    // enviar email inline...
    var smtp = new SmtpClient("mail.server.com");
    smtp.Send(new MailMessage("[email protected]", pedido.Email,
        "Pedido processado", $"Total: {pedido.Total}"));

    conn.Execute("UPDATE Pedidos SET Total=@t, Status=@s WHERE Id=@id",
        new { t = pedido.Total, s = pedido.Status, id = pedidoId });
}
// Banco, regra de negócio e email no mesmo lugar
Sinal de alerta: Impossível testar, impossível trocar o banco ou o provedor de e-mail sem mudar esse método. Acoplamento total.
Avançado Aplica SOLID conscientemente — cada classe tem uma razão para mudar ▼
exemplo de código
C#
// Open/Closed Principle — adicionar desconto sem modificar código existente
public interface ICalculadorDesconto
{
    bool Aplica(Pedido pedido);
    decimal Calcular(Pedido pedido);
}

public class DescontoVip : ICalculadorDesconto
{
    public bool Aplica(Pedido p) => p.Cliente.IsVip;
    public decimal Calcular(Pedido p) => p.Total * 0.20m;
}

public class DescontoPromo : ICalculadorDesconto
{
    public bool Aplica(Pedido p) => p.CriadoEm.Month == 11; // novembro
    public decimal Calcular(Pedido p) => p.Total * 0.10m;
}

// Handler usa DIP — depende de abstrações
public class ProcessarPedidoHandler
{
    private readonly IPedidoRepository _repo;
    private readonly IEnumerable<ICalculadorDesconto> _descontos;
    private readonly INotificador _notificador;

    public ProcessarPedidoHandler(
        IPedidoRepository repo,
        IEnumerable<ICalculadorDesconto> descontos,
        INotificador notificador)
    {
        _repo = repo;
        _descontos = descontos;
        _notificador = notificador;
    }

    public async Task Handle(ProcessarPedidoCommand cmd, CancellationToken ct)
    {
        var pedido = await _repo.GetByIdAsync(cmd.PedidoId, ct);
        var desconto = _descontos.FirstOrDefault(d => d.Aplica(pedido));
        if (desconto != null) pedido.AplicarDesconto(desconto.Calcular(pedido));
        pedido.Processar();
        await _repo.SaveAsync(pedido, ct);
        await _notificador.NotificarAsync(pedido, ct);
    }
}
// Novo tipo de desconto = nova classe, zero mudança no handler
Marco: Entende que SOLID não é um checklist, é um guia para minimizar o custo de mudança. Questiona quando vale a complexidade adicional.

// Arrange-Act-Assert com Moq
public class ProcessarPedidoHandlerTests
{

    private readonly Mock<INotificador> _notifMock = new();
    private readonly ProcessarPedidoHandler _handler;

    public ProcessarPedidoHandlerTests()
    {
        _handler = new ProcessarPedidoHandler(
            _repoMock.Object, [], _notifMock.Object);
    }

    [Fact]
    public async Task Handle_PedidoValido_DeveSalvarENotificar()
    {
        // Arrange
        var pedido = PedidoBuilder.Valido().Build();

                 .ReturnsAsync(pedido);

        // Act
        await _handler.Handle(new ProcessarPedidoCommand(pedido.Id), default);

        // Assert
        _repoMock.Verify(r => r.SaveAsync(pedido, default), Times.Once);
        _notifMock.Verify(n => n.NotificarAsync(pedido, default), Times.Once);

    public async Task Handle_PedidoNaoEncontrado_DeveLancarExcecao()

            () => _handler.Handle(new ProcessarPedidoCommand(Guid.NewGuid()), default));

}

// REST semântico com versionamento e ProblemDetails
[ApiController]
[ApiVersion("2.0")]
[Route("api/v{version:apiVersion}/pedidos")]
public class PedidosController : ControllerBase
{
    [HttpGet("{id:guid}")]

        Guid id, CancellationToken ct)
    {

        return pedido is null ? NotFound() : Ok(PedidoResponse.From(pedido));

    [ProducesResponseType<ValidationProblemDetails>(StatusCodes.Status422UnprocessableEntity)]

// Event Sourcing — estado derivado de eventos imutáveis
public class PedidoAggregate
{
    private readonly List<IDomainEvent> _eventos = new();
    public Guid Id { get; private set; }
    public StatusPedido Status { get; private set; }
    public decimal Total { get; private set; }

    // Reconstitui estado a partir de eventos (replay)
    public static PedidoAggregate From(IEnumerable<IDomainEvent> eventos)
    {
        var aggregate = new PedidoAggregate();
        foreach (var evento in eventos)
            aggregate.Apply(evento);
        return aggregate;
    }

    public void Fechar()
    {
        if (Status != StatusPedido.Aberto)
            throw new DomainException("Pedido não está aberto.");
        var evento = new PedidoFechadoEvent(Id, DateTime.UtcNow);
        Apply(evento);
        _eventos.Add(evento); // persiste o evento, não o estado
    }

    private void Apply(IDomainEvent e)
    {
        switch (e)
        {
            case PedidoFechadoEvent:
                Status = StatusPedido.Fechado;
                break;
            case ItemAdicionadoEvent item:
                Total += item.Subtotal;
                break;
        }
    }
}
// Audit log gratuito. Time travel. Projeções customizadas.
Especialista: Sabe que Event Sourcing resolve auditoria e time travel, mas adiciona complexidade. Decide com base em requisitos reais, não hype. Discute eventual consistency, snapshots, e projeções.

// Cache distribuído com Redis + Stampede protection
public async Task<T?> GetOrSetAsync<T>(
    string key, Func<Task<T>> factory, TimeSpan ttl)
{
    var cached = await _redis.GetAsync(key);
    if (cached != null) return Deserialize<T>(cached);

    // Probabilistic early expiration — evita thundering herd
    await _semaphore.WaitAsync();
    try
    {
        cached = await _redis.GetAsync(key); // double-check
        if (cached != null) return Deserialize<T>(cached);

        var value = await factory();
        await _redis.SetAsync(key, Serialize(value), ttl);
        return value;
    }
    finally { _semaphore.Release(); }
}

Intermediário Parameterized queries, hash de senha, JWT correto ▼
exemplo de código
C#
// Autenticação segura
public async Task<string?> AutenticarAsync(string login, string senha)
{
    // Parameterized query — impossível injetar
    var usuario = await _conn.QuerySingleOrDefaultAsync<Usuario>(
        "SELECT * FROM Usuarios WHERE Login = @login",
        new { login });

    if (usuario == null) return null;

    // BCrypt com work factor adequado (12+)
    if (!BCrypt.Net.BCrypt.Verify(senha, usuario.SenhaHash))
        return null;

    // JWT com claims mínimos necessários
    var claims = new[]
    {
        new Claim(JwtRegisteredClaimNames.Sub, usuario.Id.ToString()),
        new Claim(JwtRegisteredClaimNames.Jti, Guid.NewGuid().ToString()),
        new Claim("role", usuario.Perfil)
    };

    var token = new JwtSecurityToken(
        issuer: _config["Jwt:Issuer"],
        audience: _config["Jwt:Audience"],
        claims: claims,
        expires: DateTime.UtcNow.AddHours(1),
        signingCredentials: _credentials);

    return new JwtSecurityTokenHandler().WriteToken(token);
}
// Nunca retorna "senha incorreta" vs "usuário não encontrado" separados
Marco: Timing attack prevention — mesma mensagem para usuário inválido e senha errada. BCrypt com custo adequado ao hardware.

async function buscarPedidos() {
  const btn = document.querySelector('#btn-buscar');
  const lista = document.querySelector('#lista-pedidos');

  btn.disabled = true;
  btn.textContent = 'Carregando...';

  try {
    const res = await fetch('/api/v1/pedidos', {
      headers: { 'Authorization': `Bearer ${getToken()}` }
    });

    if (!res.ok) throw new Error(`HTTP ${res.status}`);

    const pedidos = await res.json();

    // textContent em vez de innerHTML — sem risco de XSS
    lista.innerHTML = '';
    pedidos.forEach(p => {
      const li = document.createElement('li');
      li.textContent = `Pedido #${p.id} — R$ ${p.total.toFixed(2)}`;
      lista.appendChild(li);
    });
  } catch (err) {
    console.error('Erro ao buscar pedidos:', err);
    lista.textContent = 'Erro ao carregar pedidos.';
  } finally {
    btn.disabled = false;
    btn.textContent = 'Buscar';
  }
}
// textContent previne XSS. Loading state melhora UX.
